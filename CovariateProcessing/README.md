# Directory Information

[**01-MakingGrids.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/01-MakingGrids.pdf) This script is used to construct a clean, continuous hexagonal analytical grid over the study area in the Puget Sound region.
  - It uses `reduced_area.gpkg`, `Dclass_cut.gpk`, and `puget_poly.gpkg` located in [**GIS_COVS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/GIS_COVS).
  - Final product is `grid2km_hex.gpkg` located in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS) and used to link static and dynamic covariates to specific locations. Grids used throughout the analysis and where density is assigned to.
  - `SS_Bathymetry.tiff` is the [Bathymetric Base Map for the Salish Sea Bioregion](https://www.arcgis.com/home/item.html?id=f23310e286614728a823ae84ab865cc1) by Aquila Flower, 2021. We used it to make a closed polygon shape for other analyses, but is not included due to size and must be downloaded directly from source if wanting to follow code line by line.

[**02-StaticCovariates.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/02-StaticCovariates.pdf) Script to calculate and assign static environmental covariates across the study area for both grid cells and surveyed tracklines.
  - It uses `reduced_area.gpkg`, `Dclass_cut.gpk`, `puget_poly.gpkg`, and `all_tracks_2026_final.gpkg` located in [**GIS_COVS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/GIS_COVS) and `grid2km_hex.gpkg` located in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS).
  - For BPI and depth, will need to download [Bathymetric Base Map for the Salish Sea Bioregion](https://www.arcgis.com/home/item.html?id=f23310e286614728a823ae84ab865cc1) directly.
  - Also uses files `pmep_estu_ext.gpkg`, `pmep_substrat_habitat.gpkg`, and `pmep_biotic_habitat.gpkg` from [**GIS_COVS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/GIS_COVS).
  - Final products are `all_static_hex.csv` and `all_static_track.csv` located in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS).

[**03-DynamicCovariates.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/03-DynamicCovariates.pdf) Is in intermediate processing of initial calculations for dynamic covariates.
  - It uses `reduced_area.gpkg`, `Dclass_cut.gpk`, `puget_poly.gpkg`, and `all_tracks_2026_final.gpkg` located in [**GIS_COVS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/GIS_COVS), `grid2km_hex.gpkg` located in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS), and the [Bathymetric Base Map for the Salish Sea Bioregion](https://www.arcgis.com/home/item.html?id=f23310e286614728a823ae84ab865cc1).
  - Additionally, [SalishSeaCast](https://salishsea.eos.ubc.ca/nemo/) model outputs must be downloaded from their [SalishSeaCast ERDDAP site](https://salishsea.eos.ubc.ca/erddap/index.html).
  - The [SalishSeaCast NEMO Model Grid](https://salishsea.eos.ubc.ca/erddap/griddap/ubcSSnBathymetryV21-08.html) was downloaded from the SalishSeaCast NEMO model (Soontiens et al, 2016; Soontiens and Allen, 2017) their ERDDAP server (https://salishsea.eos.ubc.ca/erddap/) on **01/20/2025** for all months from 2012 to end of 2024.
      - Will need Chemistry Fields [ubcSSg3DChemistryFields1moV21-11](https://salishsea.eos.ubc.ca/erddap/info/ubcSSg3DChemistryFields1moV21-11/index.html)
      - Biology Fields [ubcSSg3DBiologyFields1moV21-11](https://salishsea.eos.ubc.ca/erddap/info/ubcSSg3DBiologyFields1moV21-11/index.html)
      - Physics Field [ubcSSg3DPhysicsFields1moV21-11](https://salishsea.eos.ubc.ca/erddap/info/ubcSSg3DPhysicsFields1moV21-11/index.html)
      - Turbidity Fields [ ubcSSg3DLightFields1moV21-11](https://salishsea.eos.ubc.ca/erddap/info/ubcSSg3DBiologyFields1moV21-11/index.html).
- Maxent Habitat Rasters: Raster files provided by from the Northwest Forest Plan’s 30-year Marbled Murrelet nesting habitat assessment (Duarte et al 2025; in review).
- Due to large size of the intermediate files, they are not provided in the **OUTPUTS** folder, but final covariate files are provided from [**06-CombineDynamicCovariates.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/06-CombineDynamicCovariates.pdf) as `dynamic_data_clean_grids.rds` and `dynamic_data_clean_tracks.rds` in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS).

[**04-TidalData.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/04-TidalData.pdf) Is in intermediate processing of initial calculations for downloading and calculating monthly tidal data as well as survey level tidal data.
  - It uses `reduced_area.gpkg`, `puget_poly.gpkg`, `all_tracks_2026_final.gpkg`, `CO-OPS_Regional_Zoning.gpkg`, and `lat_long_times.rds` located in [**GIS_COVS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/GIS_COVS), `grid2km_hex.gpkg` located in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS).
  - `CO-OPS_Regional_Zoning.gpkg` is the [Zoning](https://www.arcgis.com/home/item.html?id=20fbff7c96fe4afebbf92e2c9a89b856) used to determine the tidal information for areas outside of the station locations.
  - Code includes all API retrevial details for each of the stations data were downloaded from.
  - Final products are intermediate and they are not provided in the **OUTPUTS** folder, but final covariate files are provided from [**06-CombineDynamicCovariates.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/06-CombineDynamicCovariates.pdf) as `dynamic_data_clean_grids.rds` and `dynamic_data_clean_tracks.rds` in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS).

[**05-AverageCurrents.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/05-AverageCurrents.pdf) Is in intermediate processing of initial calculations for downloading and calculating monthly tidal data as well as survey level tidal data.
  - It uses `reduced_area.gpkg`, `puget_poly.gpkg`, `puget_current_stations.csv`, and `all_tracks_2026_final.gpkg` located in [**GIS_COVS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/GIS_COVS), `grid2km_hex.gpkg` located in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS), and all files located in the [*CURRENT zipfile*](https://github.com/sjgillman/PS_MAMU/blob/ee2bd36740f78e3e8171e450a4e2bd19f8a5e3a7/CovariateProcessing/GIS_COVS/CURRENTS.zip).
  - The [*CURRENT zipfile*](https://github.com/sjgillman/PS_MAMU/blob/ee2bd36740f78e3e8171e450a4e2bd19f8a5e3a7/CovariateProcessing/GIS_COVS/CURRENTS.zip) has all csv files that were downloaded individually for all stations from 2015 -2017 for a total of 134 stations. Recorded information were available at varying depths for each station and length and time of recordings also varied per station. It also includes `SSM_mesh.gpkg` which was made with **QGIS** from the [Salish Sea Model was developed by Pacific Northwest National Laboratory (PNNL)](https://ecology.wa.gov/research-data/data-resources/models-spreadsheets/modeling-the-environment/salish-sea-modeling). The original file is a `geodatabase` file located on the [SSM's Downloadable Files Site](https://fortress.wa.gov/ecy/ezshare/EAP/SalishSea/SalishSeaModelBoundingScenarios.html) under the *Geographical Information System files* section.
  - Final products are `grid_currents.csv` and `track_currents.csv` in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS) and are used in [**DataAnalysis**](https://github.com/sjgillman/PS_MAMU/blob/main/DataAnalysis).
  - *NOTE* average current speed was eventually dropped from the model due to correlation, but code it kept for records.


[**06-CombineDynamicCovariates.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/06-CombineDynamicCovariates.pdf) Final script for combining dynamic covariates together from [**03-DynamicCovariates.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/03-DynamicCovariates.pdf) and [**04-TidalData.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/04-TidalData.pdf).
  - It uses `all_tracks_2026_final.gpkg` and `lat_long_times.rds` located in [**GIS_COVS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/GIS_COVS), `grid2km_hex.gpkg` located in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS), and intermediate files created in [**03-DynamicCovariates.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/03-DynamicCovariates.pdf) and [**04-TidalData.pdf**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/04-TidalData.pdf).
 - Final products are `dynamic_data_clean_grids.rds` and `dynamic_data_clean_tracks.rds` in [**OUTPUTS**](https://github.com/sjgillman/PS_MAMU/blob/main/CovariateProcessing/OUTPUTS) and are used in [**DataAnalysis**](https://github.com/sjgillman/PS_MAMU/blob/main/DataAnalysis).

## GIS_COVS

These geopackages were created via QGIS or directly in R either as stand alone objects (e.g., study area polygon, Puget Sound polygon) or from files linked with details on their location. If the geopackage is modified, I used QGIS to crop the spatial object to within the study area and converted the projections to CRS 26910 for easier use in R and for reducing the file size. All Original sources for each geopackage are linked. More detail as to when and how they are used can be found within the code.
